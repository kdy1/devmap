# git

협업을 하려면 여러 명이 작성한 소스코드를 한곳에 저장해야합니다.
소스코드를 관리해주는 도구중에 가장 널리 쓰이는 건 `git`이고, 이 문서에서는 `git`의 기본 개념을 소개합니다.

## 레포지토리

레포지토리 초기화 명령어인 `git init`을 제외한 모든 `git` 명령어는 레포지토리 내부에서만 작동합니다.
레포지토리는 기본적으로는 `git`이 관리할 소스코드의 범위입니다.

한 프로젝트에 웹사이트 / 서버 / 모바일 앱 같은 식으로 레포지토리가 여러 개인 경우도 있고 한 프로젝트에 관련된 모든 소스코드가 한 레포지토리에 들어있는 경우도 있습니다.
그리고 후자가 꽤나 많이 쓰이기 때문에 따로 부르는 이름도 있는데요, 후자 같은 형태를 모노레포(Monolithic repository)라고 부릅니다.

## 브랜치

`git`에서는 브랜치의 개념이 매우 중요합니다.
아주 많은 동작이 브랜치를 필요로 하기 때문에, 레포지토리를 초기화했을 때 비어있는 브랜치가 생성됩니다.
브랜치는 소스코드의 **변경사항**을 저장하는 공간인데요, (작업 과정에서의 자잘한 변경을 제외한) 모든 소스코드가 한 브랜치에 저장되어있는 경우가 대부분입니다.
**변경사항**을 저장하는 공간인데 **전체 내용**이 들어있는 건, 비어있는 브랜치에 내용을 추가하는 것**도** 변경이기 때문입니다.

브랜치에 내용을 추가하는 걸 커밋(`git commit`)이라고 합니다.
커밋한 내용은 브랜치에 저장되는데요, 주의하셔야 할 게 커밋한 내용은 로컬에만 저장된다는 것입니다.
